# Map Memory 產品規則

更新日期：2026-05-13

## 產品定位

Map Memory 是家人／好友共用的地圖回憶工具，用來記錄：

- 想去的地點
- 去過後還想再去的地點
- 去過但只想留在回憶中的地點
- 每次拜訪的回憶紀錄、照片、喜歡程度與文字筆記

主要目標不是做一般地圖工具，而是做「以地點為核心的回憶時間軸」。

---

## 地點狀態

地點狀態固定為三種：

### 1. 想去

代表還沒去過。

顯示邏輯：

- icon 使用實心愛心
- 狀態文字：想去
- 適合尚未拜訪、想收藏的地點

### 2. 還想去

代表已經去過，而且仍然推薦或想再去。

顯示邏輯：

- icon 使用空心愛心＋閃亮星星
- 狀態文字：還想去
- 新增第一次回憶後，若原本狀態是「想去」，系統會自動轉成「還想去」

### 3. 回憶中

代表已去過，但不推薦或不想再去。

顯示邏輯：

- icon 使用半透明／玻璃感愛心概念
- 狀態文字：回憶中

---

## 地點資料規則

每個地點包含：

- 地點名稱
- 狀態
- 地址
- 導航目的地
- 喜愛程度
- 標籤
- 地點筆記
- 經緯度
- 地點封面照片
- 回憶時間軸

### 地點照片規則

地點照片目前限制為：

- 最多 1 張
- 有照片時不能再上傳新照片
- 必須先刪除原照片，才能重新上傳
- 這張照片就是地點封面照片

上傳前必須自動壓縮：

- 長邊最大 1600px
- 轉成 WebP
- 品質 0.75
- 目標是降低 Firebase Storage 成本與加快手機載入速度

---

## 回憶紀錄規則

每個地點可以有多筆回憶紀錄。

每筆回憶包含：

- 拜訪日期
- 文字紀錄
- 喜歡程度
- 回憶照片
- 建立時間
- 更新時間

### 日期規則

- 回憶日期不能是未來日期
- 日期最多只能選今天或今天以前
- 不符合邏輯的未來日期必須阻擋

### 回憶照片規則

每筆回憶照片：

- 最多 10 張
- 可新增照片
- 可刪除單張照片
- 可點擊放大預覽
- 可左右切換
- 上傳前應逐步導入壓縮機制，避免照片過大造成成本增加

### 回憶排序

回憶時間軸排序：

- 日期新到舊
- 最新拜訪顯示在上方

---

## 回憶刪除規則

每筆回憶右上角提供：

- 編輯按鈕
- 刪除按鈕

刪除回憶後：

- visitCount 自動重算
- firstVisitedAt 自動重算
- lastVisitedAt 自動重算
- 若已沒有任何回憶，firstVisitedAt / lastVisitedAt 需清除

---

## 回憶編輯規則

每筆回憶可編輯：

- 日期
- 文字
- 喜歡程度
- 照片新增
- 照片刪除

編輯後：

- 時間軸需即時刷新
- visitCount 不變
- firstVisitedAt / lastVisitedAt 需重新計算
- 不可造成 React 無限重新渲染

---

## 地圖群規則

目前支援地圖群：

- 可建立地圖群
- 可使用邀請碼申請加入
- 申請加入需原群主同意
- 可退出地圖群
- 群主可刪除地圖群

地圖群資料依 Firebase groups 儲存。

---

## Firebase Storage 暫時規則

目前開發階段曾使用公開測試規則：

```txt
allow read, write: if true;
```

這只適合開發階段。

正式上線前必須改成：

- 登入後才能讀寫
- 只能存取自己／自己群組的資料
- 限制檔案大小
- 限制檔案類型
- 避免陌生人惡意上傳

---

## 重要產品原則

- 操作要適合手機
- 介面要像回憶 App，不是後台系統
- 地圖瀏覽與地點清單的回憶呈現方式要一致
- 盡量避免使用者需要理解技術細節
- 新功能每次只做小步快跑，避免改壞既有功能
