## 今日完成項目

### 1. Firebase Storage 權限

遇到錯誤：

```txt
Firebase Storage: User does not have permission
storage/unauthorized
```

處理方式：

- 到 Firebase Console
- Storage
- Rules
- 開發階段暫時改成公開測試規則
- Publish 後 Ctrl + F5
- 照片可正常上傳

注意：正式上線前不可維持公開規則。

- 照片可點擊放大
- 多張可左右切換

### 3. React Minified error #185 修正

曾在 MapView 與 PlaceListView 遇到：

```txt
Maximum update depth exceeded
React Minified error #185
```

修正方向：

- 移除容易造成循環的 useEffect setState
- detailPlace 改用 detailPlaceId + useMemo 從 places 找最新資料
- sortedVisits 用 useMemo
- filter 狀態更新先比對是否相同
- handlers 用 useCallback

---

### 4. 回憶新增後即時刷新

地點清單點開卡片後新增回憶，詳細視窗需即時顯示新回憶。

目前做法：

- 不把完整 detailPlace 固定存在 state
- 只存 detailPlaceId
- 每次 places 更新後用 useMemo 找最新 place

---

### 5. 回憶刪除功能

- visitCount 自動更新
- firstVisitedAt / lastVisitedAt 自動重算
- 若無回憶，清除 firstVisitedAt / lastVisitedAt

### 6. 回憶編輯功能

已加入：

- 每筆回憶右上角 📝
- 可修改日期
- 可修改文字
- 可修改喜愛程度
- 可新增照片
- 可刪除照片
- 日期不可是未來日期

---

### 7. 回憶卡片版型

最新方向：

地圖瀏覽與地點清單需一致。

版型：

```txt
左側：日期、喜歡程度、文字
右側：照片
```

照片：

- 顯示 2 張
- 第 2 張顯示 +N
- 照片固定在框內
- 中間要有縫隙
- 不可蓋到右上角 📝 / 🗑️

文字：

- 超過寬度自動換行
- 超過高度可滾動查看

---

### 8. 地點資訊區塊

回憶彈窗上方需顯示：

- 狀態
- 喜愛程度
- 標籤
- 地點筆記

目前要求：

- 不要有太多空白
- 資訊要壓縮排列
- 地點筆記可限制高度並滾動

---

### 9. 地點照片限制

新增 / 編輯地點照片目前改為：

- 最多 1 張
- 可刪除
- 刪除後可重新上傳
- 已有照片時不能直接再上傳
- 該照片作為地點封面

## 今日產生與覆蓋過的主要檔案

### app/page.tsx

用途：

- 傳遞 MapView 所需 props
- 管理新增 / 編輯 / 刪除回憶
- 管理 Firebase updateDoc
- 管理 VisitFormModal create / edit mode

注意：

MapView 若需要 onEditVisit / onDeleteVisit，page.tsx 必須傳入：

```tsx
onEditVisit={openEditVisitModal}
onDeleteVisit={handleDeleteVisit}
```

### components/MapView.tsx

用途：

- 地圖 marker
- 地圖搜尋
- 地圖卡片
- 點卡片開地圖回憶時間軸
- 地圖回憶彈窗
- 照片預覽

注意：

- 無回憶時，點地圖卡片不反應
- 有回憶時，才開啟時間軸

### components/PlaceListView.tsx

用途：

- 地點清單
- 地點清單搜尋
- 地點詳情
- 回憶時間軸
- 回憶編輯 / 刪除
- 照片預覽

注意：

- 若覆蓋後沒效果，先搜尋「地點筆記」確認是否為新版

### components/VisitFormModal.tsx

用途：

- 新增回憶
- 編輯回憶
- 上傳回憶照片
- 刪除回憶照片

### components/PlaceFormModal.tsx

用途：

- 新增地點
- 編輯地點
- 上傳地點照片
- 刪除地點照片
- 地點照片壓縮
