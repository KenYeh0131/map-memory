## 已完成：地點清單回憶版型

- 上方地點資訊區塊
- 狀態 / 喜愛程度 / 標籤 / 地點筆記

- 文字超過高度可滑動

---

## 已完成：地圖瀏覽回憶版型

地圖瀏覽點擊 marker 後會展開卡片。

點擊卡片本體：

- 若有回憶，打開地圖回憶時間軸
- 若無回憶，不反應

地圖回憶需要與地點清單維持一致版型。

---

## 已完成：地點照片限制與壓縮

新增 / 編輯地點照片：

- 最多 1 張
- 可刪除
- 刪除後可重新上傳
- 上傳前自動壓縮
- 長邊最大 1600px
- WebP
- 品質 0.75

---

## 近期下一步建議

### 1. 回憶照片也加入壓縮

目前地點照片已壓縮，但回憶照片仍建議補上同樣邏輯。

建議規則：

- 長邊最大 1600px
- WebP
- 品質 0.7～0.8
- 每筆回憶最多 10 張

優先度：高

---

### 2. Storage 安全規則

目前開發階段可能仍使用公開規則。

正式上線前需改：

- 登入後才能上傳
- 僅允許自己群組資料
- 限制圖片大小
- 限制 MIME type
- 限制路徑

優先度：高

---

### 3. 地點適合時間 / 季節功能

需求：

- 地點可設定適合月份或季節
- 當目前時間符合，地圖 marker 顯示提示
- 不符合時，卡片顯示「未達建議時間」

優先度：中

---

### 4. 回憶時間軸整體頁

可新增獨立頁面：

- 依日期顯示所有地點回憶
- 像社群動態牆
- 可篩選地點、標籤、月份

優先度：中

---

### 5. PWA 與手機體驗

已做 PWA icon 相關設定，後續可補：

- 安裝提示
- 離線提示
- 啟動畫面
- 手機拍照上傳流程優化

優先度：中

---

### 6. 成本控制

目前主要成本來源：

- Google Maps API
- Firebase Storage
- Firebase Firestore

已採取：

- 地點照片單張限制
- 地點照片壓縮

待補：

- 回憶照片壓縮
- 圖片大小限制
- Firebase Storage rules
- 圖片刪除時同步刪 Storage 檔案

優先度：中高

---

## 開發原則

- 每次改一小段功能
- 改完先測試
- 不要一次大重構
- 若 GPT 視窗太長，使用 docs 紀錄接續
- Cursor Agent 開新對話前，先讀 docs
- 不熟程式時，優先使用完整覆蓋檔，不要手動找片段
