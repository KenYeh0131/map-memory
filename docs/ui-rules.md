# Map Memory UI 規則

更新日期：2026-05-13

## 整體設計原則

- 手機優先
- 按鈕要大、好點
- 視覺要像旅遊回憶 App
- 地圖瀏覽與地點清單的回憶卡片版型要一致
- 不要做成後台表格感
- 重要操作要能一眼看懂

---

## 底部導覽

目前底部導覽：

- 地圖瀏覽
- 地點清單
- 我的／設定

地圖瀏覽與地點清單都需要能看到回憶時間軸。

---

## 地點卡片

地點卡片顯示：

- 封面照片
- 地點名稱
- 地址
- 狀態
- 標籤
- 拜訪次數
- 最近拜訪
- 喜愛程度
- 操作按鈕

操作按鈕：

- 出發
- ＋回憶
- 編輯
- 刪除

---

## 地點狀態視覺

### 想去

- 實心愛心
- 紅色系

### 還想去

- 空心愛心＋閃亮星星
- 橘色系

### 打卡完成

- 半透明玻璃感愛心概念
- 灰階／霧面感

---

## 地圖 Marker 規則

使用者不喜歡「地圖地標＋愛心」的組合。

可接受：

- 實心地標
- 地標中間空心部分做成愛心形狀

目前 MapView marker 使用 SVG 方式產生。

---

## 地圖瀏覽搜尋

地圖搜尋採展開／收合模式。

搜尋內包含：

- 狀態篩選：想去／還想去／打卡完成
- 關鍵字搜尋
- 愛心數篩選
- 標籤篩選

狀態按鈕需同一行 3 個按鈕。

---

## 地點清單搜尋

地點清單搜尋需跟地圖瀏覽一致：

- 展開／收合模式
- 狀態篩選同一行
- 關鍵字
- 愛心數
- 標籤

---

## 回憶時間軸彈窗

點擊地點卡片可打開詳情。

### 地點資訊區塊

上方需顯示地點設定資訊：

- 狀態
- 喜愛程度
- 標籤
- 地點筆記

版面要求：

- 不要太多空白
- 狀態、喜愛程度、標籤盡量壓縮排列
- 地點筆記可限制高度並捲動
- 手機上不要浪費可視高度

---

## 回憶卡片版型

地圖瀏覽與地點清單需相同。

回憶卡片採左右分區：

```txt
左側：日期、喜歡程度、文字
右側：照片
```

### 左側文字區

- 日期與喜歡程度同一行
- 文字可自動換行
- 超過高度時，文字區可上下滑動
- 文字不得把照片區擠壞

### 右側照片區

目前版型：

- 顯示 2 張照片
- 第 2 張若還有更多照片，顯示 +N
- 兩張照片中間必須有縫隙
- 照片限制在固定框內
- 照片不得超出卡片
- 照片高度不可蓋到右上角編輯／刪除按鈕

### 編輯與刪除

每筆回憶右上角：

- 📝 編輯
- 🗑️ 刪除

---

## 照片預覽

點回憶照片可放大。

預覽功能：

- 黑色遮罩
- 點 X 關閉
- 多張照片可左右切換
- 顯示目前第幾張 / 總張數

---

## 新增 / 編輯回憶彈窗

VisitFormModal 支援：

- create 模式
- edit 模式

可編輯：

- 拜訪日期
- 回憶文字
- 喜愛程度
- 照片新增
- 照片刪除

日期限制：

- 不能選未來日期

---

## 新增 / 編輯地點彈窗

PlaceFormModal 目前規則：

- 地點照片最多 1 張
- 已有照片時不能再新增
- 必須刪除原照片後才能重新上傳
- 上傳前自動壓縮照片
- 選擇檔案文字必須清楚可見，不能太淡

照片壓縮：

- 長邊最大 1600px
- WebP
- 品質 0.75

---

## 目前已知 UI 注意事項

- MapView 若加入新 props，page.tsx 必須同步傳入
- 地圖瀏覽與地點清單若共用同版型，需確認兩邊同時更新
- 若下載檔案覆蓋失敗，先用關鍵字搜尋確認版本，例如「地點筆記」
